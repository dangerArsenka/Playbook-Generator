Репозиторій має таку логічну структуру:
– main.py – основний файл Flask-застосунку, у якому реалізовано маршрути вебінтерфейсу, обробку сценарію кіберінциденту, виклик модулів аналізу, збереження результатів та експорт PDF-звіту.
– settings.py – файл налаштувань системи, у якому визначаються параметри роботи застосунку, шляхи до бази даних і директорій, а також конфігурація зовнішніх API-ключів.
– language.py – модуль визначення мови введеного сценарію, який використовується для формування відповіді українською або англійською мовою відповідно до мови запиту користувача.
– ioc_extractor.py – модуль виявлення індикаторів компрометації у тексті сценарію. Виконує пошук IP-адрес, доменів, URL-адрес і хеш-значень файлів, а також фільтрацію частини хибних збігів.
– threat_intel/ – каталог модулів Threat Intelligence аналізу. Містить агент збагачення індикаторів компрометації та окремі компоненти для взаємодії із зовнішніми джерелами перевірки IOC. Результати роботи цього каталогу використовуються для уточнення контексту інциденту.
– incident_analysis.py – модуль AI Incident Analysis Agent, який виконує оцінку сценарію інциденту із застосуванням генеративної моделі, визначає орієнтовний рівень критичності, рівень впевненості, обґрунтування та можливі відповідності MITRE ATT&CK.
– playbook_generator.py – модуль формування підсумкового плейбука реагування. На основі сценарію, результатів IOC-аналізу, Threat Intelligence контексту та AI-оцінки генерує структурований плейбук у форматі Markdown і Mermaid-діаграму.
– database.py – модуль роботи з локальною базою даних SQLite. Забезпечує створення таблиць, збереження результатів аналізу, отримання історії, перегляд окремих записів та видалення збережених результатів.
– pdf_report.py – модуль формування PDF-звіту за результатами аналізу. До звіту включаються дані про сценарій інциденту, оцінку критичності, результати Threat Intelligence аналізу, відповідності MITRE ATT&CK, сформований плейбук і код Mermaid-діаграми.
– templates/ – каталог HTML-шаблонів вебінтерфейсу. Містить сторінки для введення сценарію, перегляду результатів аналізу, історії збережених інцидентів і окремого збереженого запису.
– static/ – каталог статичних файлів вебінтерфейсу. Містить CSS-стилі, JavaScript-файли та допоміжні ресурси для візуального оформлення сторінок і відображення Mermaid-діаграм.
– outputs/ – каталог для збереження сформованих PDF-звітів.
– requirements.txt – перелік Python-бібліотек, необхідних для запуску програмного засобу.
– .env.example – приклад конфігураційного файлу для налаштування API-ключів і параметрів середовища.
Розміщення повного коду в репозиторії дозволяє не перевантажувати основний текст дипломної роботи великими лістингами, але забезпечує можливість перевірки програмної реалізації

Як запустити програму:
python -m venv venv - створення нового віртуального середовища
venv\Scripts\activate - активація віртуального середовища у Windows
pip install -r requirements.txt - встановлення необхідних бібліотек
copy .env.example .env - створення файлу .env на основі прикладу
notepad .env - відкриття файлу .env для додавання API-ключів
python main.py - запуск Flask-застосунку
http://127.0.0.1:5000 - адреса для відкриття вебінтерфейсу в браузері
